ğŸ—ï¸ Terraform Infrastructure

The Terraform configuration provisions a Builder EC2 instance with Docker pre-installed.

Resources Created

Resource Description
aws_instance EC2 instance (t3.medium) for running Docker workloads
aws_security_group Security group with SSH (22) and App (5001) access
aws_key_pair SSH key pair for instance access
tls_private_key Auto-generated RSA key pair

Deploy Infrastructure

cd terraform

# Initialize Terraform
terraform init

# Preview changes
terraform plan

# Apply configuration
terraform apply

Configure Variables

Update variables.tf with your values:

variable "vpc_id" {
  default = "vpc-xxxxxxxxx"  # Your VPC ID
}

variable "my_ip" {
  default = "x.x.x.x/32"  # Your public IP
}

variable "public_subnet_id" {
  default = "subnet-xxxxxxxxx"  # Your subnet
}

ğŸ”„ CI/CD Pipeline

The Jenkins pipeline automates building, testing, and pushing Docker images.

Pipeline Stages

Stage Description
Clone Repository Clones the GitHub repository
Linting Runs Flake8 (Python) and Hadolint (Dockerfile)
Security Scan Runs Bandit (Python) and Trivy (filesystem scan)
Build Docker Image Builds the Docker image with build number tag
Push to Docker Hub Pushes image to Docker Hub registry

Jenkins Setup

  1. Install Jenkins on your Builder EC2 instance
  2. Add DockerHub credentials in Jenkins:
    • ID: dockerhub-username (Secret text)
    • ID: dockerhub-password (Secret text)
  3. Create a new Pipeline job pointing to this repository
  4. The pipeline will automatically use the Jenkinsfile
